# Penjelasan Error P9 dan Perbandingan dengan Fix SQLite

Dokumen ini menjelaskan kenapa project `p9` error saat `npm start`, bagaimana alur aplikasi `p9`, dan apa bedanya dengan folder `fix sqlite`.

## Ringkasan Singkat

Error di `p9` bukan berasal dari kode layar seperti `App.js`, `Beranda`, `Pesanan`, atau SQLite.

Error utamanya adalah dependency lokal di `node_modules` tidak lengkap atau rusak.

Pesan error pentingnya:

```text
Cannot find module './generated/ParserVisitorKeys'
```

File itu dicari oleh package:

```text
hermes-parser
```

`hermes-parser` dipakai oleh React Native/Babel saat Metro melakukan bundling.

Jadi masalahnya terjadi sebelum aplikasi benar-benar jalan di HP. Metro belum sempat menjalankan logic aplikasi, karena proses compile JavaScript sudah gagal dulu.

## Kenapa P9 Error

Saat menjalankan:

```bash
npm start
```

Expo berhasil start:

```text
Starting Metro Bundler
Metro waiting on exp://...
```

Tetapi saat Android mulai bundling, muncul error:

```text
Android Bundling failed
Error: [BABEL] ... Cannot find module './generated/ParserVisitorKeys'
```

Urutan errornya kira-kira seperti ini:

```text
Expo start
Metro bundler
Babel transform
React Native codegen
hermes-parser
file internal ParserVisitorKeys tidak ditemukan
bundling gagal
```

Artinya:

- Expo server bisa hidup.
- QR code bisa muncul.
- Tapi aplikasi gagal dibundle untuk Android.
- Penyebabnya package internal `hermes-parser` tidak lengkap di `node_modules`.

Di `package-lock.json`, dependency seperti `hermes-parser`, `@react-native/codegen`, dan `@react-native/babel-preset` tercatat.

Tapi saat dicek di folder `node_modules`, package yang dibutuhkan tidak lengkap. Contohnya:

```text
node_modules/hermes-parser
```

tidak tersedia dengan benar.

Itulah kenapa Babel mencari file:

```text
node_modules/hermes-parser/dist/generated/ParserVisitorKeys
```

tetapi tidak menemukannya.

## Penyebab Yang Paling Mungkin

Penyebab paling masuk akal:

1. `npm install` pernah terhenti di tengah jalan.
2. `node_modules` tidak lengkap.
3. Ada package yang tercatat di lockfile, tapi folder aslinya tidak ada.
4. Metro memakai cache lama yang mengarah ke package yang sudah tidak utuh.

Ini bukan error SQLite.

Ini juga bukan error AsyncStorage.

Error muncul di tahap build tool, bukan di tahap aplikasi membaca database.

## Jika Nanti Mau Diperbaiki

Dokumen ini hanya menjelaskan, tidak melakukan fix.

Kalau nanti mau memperbaiki, biasanya langkahnya adalah install ulang dependency:

```powershell
Remove-Item -Recurse -Force node_modules
Remove-Item -Force package-lock.json
npm install
npx expo start -c
```

Kalau tidak mau hapus lockfile, bisa coba lebih aman dulu:

```powershell
Remove-Item -Recurse -Force node_modules
npm install
npx expo start -c
```

Tapi perintah di atas belum dijalankan di project ini karena permintaan sekarang hanya minta penjelasan, bukan fix.

## Gambaran Keseluruhan Project P9

Project `p9` adalah aplikasi pre-order dengan 4 tab utama:

1. `Toko Miiko`
2. `Admin Produk`
3. `Pesanan Masuk`
4. `Pesanan Selesai`

### 1. Toko Miiko

Tab ini berisi form input pesanan.

User bisa mengisi:

- nama customer
- produk
- jumlah
- tanggal ambil

Data pesanan yang dibuat berisi:

```js
{
  id,
  customer,
  items,
  total,
  date,
  status: "Menunggu",
  isPO: true
}
```

Satu pesanan bisa punya banyak item produk.

### 2. Admin Produk

Tab ini untuk mengelola produk.

Fitur yang ada:

- tambah produk
- edit produk
- hapus produk
- validasi nama produk agar tidak duplikat

Produk yang ditambahkan akan muncul di dropdown saat membuat pesanan.

### 3. Pesanan Masuk

Tab ini menampilkan pesanan dengan status:

```text
Menunggu
```

Fitur yang ada:

- lihat daftar pesanan masuk
- edit nama customer
- edit tanggal ambil
- hapus pesanan
- tandai pesanan selesai

Catatan: di `p9`, edit pesanan lebih sederhana. Yang diedit hanya nama dan tanggal. Item produk di dalam pesanan tidak diedit dari modal ini.

### 4. Pesanan Selesai

Tab ini menampilkan pesanan dengan status:

```text
Selesai
```

Data pesanan selesai dikelompokkan berdasarkan tanggal.

## Sistem Penyimpanan di P9

`p9` memakai dua jenis penyimpanan:

1. SQLite
2. AsyncStorage

SQLite dipakai untuk data utama:

- produk
- pesanan

File database:

```text
lib/database.js
```

Nama database:

```text
apk_po.db
```

Tabel yang dipakai:

```text
products
```

```text
orders
```

AsyncStorage masih dipakai untuk:

- membaca data lama dari key lama
- menyimpan tema light/dark

Key AsyncStorage yang masih ada:

```js
const THEME_KEY = "@uts_po_theme";
const LEGACY_ORDER_KEY = "@uts_po_pesanan";
const LEGACY_PRODUCT_KEY = "@uts_po_produk";
```

Jadi `p9` bukan murni SQLite.

Lebih tepatnya:

```text
p9 = SQLite untuk data produk/pesanan + AsyncStorage untuk tema dan migrasi data lama
```

## Cara P9 Memuat Data

Saat aplikasi dibuka, `App.js` melakukan langkah ini:

1. Baca pesanan lama dari AsyncStorage.
2. Baca produk lama dari AsyncStorage.
3. Baca tema dari AsyncStorage.
4. Jalankan `initializeDatabase`.
5. Kalau SQLite masih kosong, isi dari data lama atau produk default.
6. Ambil produk dari SQLite.
7. Ambil pesanan dari SQLite.
8. Tampilkan aplikasi setelah `isHydrated` menjadi `true`.

Makanya ada loading screen:

```text
Memuat aplikasi...
```

Loading ini dibuat agar UI tidak tampil sebelum data selesai dibaca.

## Tema Light dan Dark di P9

Salah satu perbedaan besar `p9` adalah adanya tema.

Di `App.js`, ada:

```js
const appThemes = {
  light: { ... },
  dark: { ... },
};
```

User bisa menekan icon di kanan header untuk mengganti tema.

Tema disimpan ke AsyncStorage:

```js
AsyncStorage.setItem(THEME_KEY, themeMode);
```

Jadi kalau app dibuka lagi, tema terakhir bisa dipakai lagi.

| Bagian | p9 | fix sqlite |

| Tujuan | Versi dengan tema, migrasi data lama, dan loading screen | Versi SQLite yang sekarang sudah ditambah tema dan loading screen, tapi tetap tanpa AsyncStorage |
| Penyimpanan produk | SQLite | SQLite |
| Penyimpanan pesanan | SQLite | SQLite |
| Penyimpanan tema | AsyncStorage | SQLite lewat tabel `settings` |
| AsyncStorage | Masih dipakai untuk tema dan membaca data lama | Tidak dipakai di source app |

## Perbedaan Penting Tentang AsyncStorage

Di `fix sqlite`, AsyncStorage sudah dibuang dari source app.

Artinya data produk, pesanan, dan tema langsung hidup di SQLite.

Di `p9`, AsyncStorage masih sengaja dipakai untuk dua hal:

1. Mengambil data lama dari versi sebelum SQLite.
2. Menyimpan preferensi tema.

Jadi kalau ada pertanyaan:

```text
Apakah p9 masih pakai AsyncStorage?
```

Jawabannya:

```text
Iya, tapi bukan untuk penyimpanan utama produk dan pesanan.
```

Produk dan pesanan utamanya sudah memakai SQLite, tetapi tema masih memakai AsyncStorage.

Di `fix sqlite`, tema tidak memakai AsyncStorage. Tema disimpan ke tabel SQLite:

```text
settings
```

Key yang dipakai:

```text
themeMode
```

Nilainya:

```text
light
```

atau:

```text
dark
```

## Perbedaan Penting Tentang Error

Error yang terjadi di `p9` bukan karena `p9` masih punya AsyncStorage.

Error juga bukan karena SQLite-nya salah.

Buktinya error terjadi di:

```text
node_modules/hermes-parser
@react-native/codegen
@react-native/babel-preset
babel-preset-expo
metro-transform-worker
```

Itu semua adalah bagian dari alat build React Native/Expo.

Kalau masalahnya dari SQLite atau AsyncStorage, biasanya errornya muncul saat app sudah berjalan, misalnya:

```text
Gagal memuat data...
Native module is null...
SQLite error...
```

Tapi error sekarang adalah:

```text
Cannot find module './generated/ParserVisitorKeys'
```

Jadi sumbernya adalah dependency build yang tidak lengkap.

## Kesimpulan

`p9` dan `fix sqlite` sekarang sama-sama punya tema light/dark dan loading screen.

Namun error yang muncul saat `npm start` bukan karena fitur aplikasinya.

Error muncul karena folder `node_modules` di `p9` tidak lengkap atau rusak, khususnya package `hermes-parser` yang dipakai Babel saat Metro melakukan bundling.

Perbedaan pentingnya: `fix sqlite` lebih bersih dari sisi storage karena tidak memakai AsyncStorage sama sekali, sedangkan `p9` masih mencampur SQLite dan AsyncStorage untuk kebutuhan migrasi serta penyimpanan tema.

Secara singkat, project `p9` mengalami error saat proses bundling (`npm start`) karena folder `node_modules` miliknya rusak atau tidak terinstal secara lengkap (khususnya modul internal `hermes-parser` yang dibutuhkan oleh Babel/Metro hilang), sedangkan project `fix sqlite` dapat berjalan lancar karena memiliki instalasi dependensi (`node_modules`) yang utuh dan bersih, serta kustomisasi tema navigasinya yang kini telah diperbaiki agar sepenuhnya kompatibel dengan standar React Navigation v7.
