# Migrasi AsyncStorage ke SQLite

Project ini sebelumnya menyimpan data dengan AsyncStorage di `App.js`.

Key yang dipakai:

- `@uts_po_pesanan` untuk daftar pesanan
- `@uts_po_produk` untuk daftar produk

Sekarang penyimpanan diganti ke SQLite lokal lewat `expo-sqlite`.

## Kenapa Tidak Cuma Ganti Import

AsyncStorage dan SQLite modelnya berbeda.

AsyncStorage menyimpan data sebagai key-value:

```js
AsyncStorage.setItem("@uts_po_produk", JSON.stringify(products));
```

Artinya seluruh daftar produk atau pesanan disimpan sebagai satu JSON besar.

SQLite menyimpan data dalam tabel. Karena itu data dipisah menjadi:

- tabel `products` untuk produk
- tabel `orders` untuk pesanan
- tabel `settings` untuk preferensi aplikasi seperti tema

Dengan struktur ini, tambah/edit/hapus data bisa dilakukan per baris, bukan menulis ulang seluruh array JSON.

## File Utama

Logic database ada di:

```text
lib/database.js
```

`App.js` sekarang hanya memanggil fungsi database seperti:

```js
initializeDatabase();
fetchProducts();
fetchOrders();
createProduct();
updateProduct();
deleteProduct();
createOrder();
updateOrder();
markOrderAsCompleted();
deleteOrder();
getSetting();
setSetting();
```

## Struktur Database

Database lokal bernama:

```text
uts_po.db
```

Tabel produk:

```sql
CREATE TABLE IF NOT EXISTS products (
  id TEXT PRIMARY KEY NOT NULL,
  name TEXT NOT NULL UNIQUE,
  price INTEGER NOT NULL
);
```

Tabel pesanan:

```sql
CREATE TABLE IF NOT EXISTS orders (
  id TEXT PRIMARY KEY NOT NULL,
  customer TEXT NOT NULL,
  items TEXT NOT NULL,
  total INTEGER NOT NULL,
  date TEXT NOT NULL,
  status TEXT NOT NULL,
  isPO INTEGER NOT NULL DEFAULT 1,
  createdAt INTEGER NOT NULL
);
```

Kolom `items` tetap disimpan sebagai JSON string karena satu pesanan bisa punya banyak item produk.

Tabel setting:

```sql
CREATE TABLE IF NOT EXISTS settings (
  key TEXT PRIMARY KEY NOT NULL,
  value TEXT NOT NULL
);
```

Tabel `settings` dipakai untuk menyimpan data kecil seperti pilihan tema `light` atau `dark`.

## Alur Saat App Dibuka

Saat aplikasi dibuka, `App.js` menjalankan:

```js
await initializeDatabase();
```

Fungsi ini membuat tabel jika belum ada. Kalau tabel produk masih kosong, produk default otomatis dimasukkan:

- Burnt Cheesecake
- Brownies
- Goguma Ppang
- Tiramisu
- Matcha Tart

Setelah itu app membaca data:

```js
const [savedProducts, savedPesanan, savedThemeMode] = await Promise.all([
  fetchProducts(),
  fetchOrders(),
  getSetting("themeMode"),
]);
```

Data dari SQLite lalu dimasukkan ke state React supaya UI tetap bisa render seperti sebelumnya.

## Dependency

AsyncStorage sudah tidak dipakai lagi di project ini. Produk, pesanan, dan pilihan tema disimpan lewat SQLite.

Dependency yang dipakai untuk storage sekarang:

```json
"expo-sqlite": "~16.0.10"
```

Di `app.json`, plugin SQLite sudah aktif:

```json
"plugins": [
  "@react-native-community/datetimepicker",
  "expo-sqlite"
]
```

Untuk web export, `metro.config.js` ditambahkan supaya Metro bisa membaca file `.wasm` dari `expo-sqlite`.

## Cara Menjalankan

Install dependency:

```bash
npm install
```

Jalankan dengan cache bersih:

```bash
npx expo start -c
```

## Catatan Sinkronisasi

SQLite ini adalah database lokal di masing-masing HP.

Kalau app dipakai di 2 HP berbeda, data tidak otomatis tersinkron. HP A dan HP B akan punya file `uts_po.db` masing-masing.

Kalau ingin data antar HP tersinkron, perlu database online atau backend seperti Firebase, Supabase, atau API sendiri.
