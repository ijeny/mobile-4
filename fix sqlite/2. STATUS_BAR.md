# Status Bar Setelah Ada Tema

Status bar adalah area sistem di paling atas layar, tempat jam, sinyal, dan baterai tampil.

Area ini tidak otomatis mengikuti warna header React Navigation. Karena itu `fix sqlite` mengatur status bar langsung dari `App.js`.

## Pengaturan di App.js

`App.js` memakai:

```js
import { StatusBar } from "react-native";
```

Lalu status bar dirender seperti ini:

```js
<StatusBar
  backgroundColor={colors.statusBar}
  barStyle={colors.statusBarStyle}
  translucent={false}
/>
```

Artinya:

- `backgroundColor={colors.statusBar}` mengambil warna status bar dari tema aktif.
- `barStyle={colors.statusBarStyle}` mengatur warna icon status bar.
- `translucent={false}` membuat status bar tidak transparan di atas layout.

## Hubungan Dengan Light/Dark Mode

Warna status bar sekarang ikut data tema di:

```text
theme/colors.js
```

Contoh:

```js
light: {
  statusBar: "#FFFFFF",
  statusBarStyle: "dark-content",
}
```

```js
dark: {
  statusBar: "#111827",
  statusBarStyle: "light-content",
}
```

> [!NOTE]
> Pada mode `light`, warna status bar diubah menjadi `#FFFFFF` (putih) dan `statusBarStyle` diset ke `"dark-content"` (ikon gelap). Hal ini dilakukan agar status bar menyatu dengan header utama yang berwarna putih dan ikon-ikon status bar tetap terlihat dengan jelas (tidak putih di atas putih yang membuatnya tidak terlihat, terutama di iOS).

Jadi saat tema diganti, status bar juga ikut berubah secara dinamis.

## Perubahan di app.json

Di Android, project ini memakai:

```json
"edgeToEdgeEnabled": false
```

Tujuannya supaya Android tidak memaksa area status bar menjadi transparan dan menabrak layout aplikasi.

## Catatan

Kalau perubahan status bar belum terlihat di HP:

```bash
npx expo start -c
```

Kalau memakai build native/dev client, perubahan `app.json` biasanya perlu build ulang aplikasi.
